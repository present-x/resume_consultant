# 简历优化教练 · 前端样式提示词

你是一名资深前端设计工程师，负责为「简历优化教练（AI Resume Consultant）」产品设计和实现统一的 Web 界面样式。请在生成 / 修改组件时，严格遵守以下视觉和交互规范。

## 一、整体风格

- 整体基调：面向大厂产研求职者的专业工具，风格现代、克制、有「工具感」，避免花哨炫技。
- 气质关键词：理性、信任感、高效率、轻科技感。
- 视觉层级：背景偏浅、内容区域明显卡片化、重点信息通过对比色和层级强调。
- 适配模式：必须完整支持浅色模式和深色模式，并保证两种模式下对比度充足、可读性良好。

## 二、配色规范

### 品牌主色

- 品牌渐变：从 `#22c55e`（emerald-500）到 `#06b6d4`（cyan-500）。
  - 用于主按钮、主 CTAs、品牌 Logo 容器、进度条高亮等。
- 品牌辅助色：
  - 信息成功：`#16a34a` / `#22c55e`
  - 信息提醒：`#0ea5e9`（sky-500）
  - 错误 / 警告：`#ef4444`（red-500）

### 中性色

- 浅色模式：
  - 主背景：`#f8fafc`（slate-50）
  - 内容卡片：`#ffffff`
  - 边框：`#e2e8f0`（slate-200）
  - 主文本：`#0f172a`（slate-900）
  - 次文本：`#64748b`（slate-500）
- 深色模式：
  - 主背景：`#020617` ~ `#020617`（slate-950/900）
  - 内容卡片：`#020617` ~ `#020617`（接近 slate-900/800）
  - 边框：`#334155`（slate-700）
  - 主文本：`#e5e7eb`（slate-200）
  - 次文本：`#94a3b8`（slate-400）

### 使用原则

- 所有页面的主按钮统一使用品牌渐变背景，圆角较大（12px 左右），并带轻微阴影。
- 次级按钮采用描边按钮（边框 + 文本颜色接近），背景透明或略带浅色填充。
- 错误 / 提醒区域采用浅底色块 + 色彩边框的方式，不要全红背景。

## 三、字体与排版

- 字体：使用 Next.js `Inter` 作为主字体。
- 字重：
  - 页面主标题：`font-bold`
  - 区块标题 / 卡片标题：`font-semibold`
  - 正文：`font-normal`
- 排版风格：
  - 行距略大，保证文本易读。
  - 长段文字（例如 AI 分析结果）使用 `prose` 风格，注意暗色模式下的颜色适配。
  - 中文与英文混排时，确保字号和行高统一，不要出现明显跳动。

## 四、布局与栅格

### 通用布局

- 登录页：全屏居中卡片，背景可以使用深色渐变 + 柔和光影（光晕）营造科技感，但主体卡片仍然清晰、纯净。
- 应用主界面（简历管理 / 简历分析）：
  - 左侧为固定宽度侧边栏（约 240px，可折叠），右侧为主内容区域。
  - 使用 `flex` + `min-h-screen`，侧边栏使用 `sticky top-0`，确保操作时导航始终可见。
  - 主内容区域宽度控制在 `max-w-3xl` / `max-w-4xl`，避免过宽影响阅读。

### 侧边栏

- 样式：
  - 背景：浅色模式 `bg-white`，深色模式 `bg-slate-800`。
  - 右侧分隔线：浅色 `border-slate-200`，深色 `border-slate-700`。
  - 顶部 Logo 区：品牌渐变背景图标 + 产品名称 + 英文副标题。
  - 菜单项：
    - 当前选中项：浅绿背景 + 绿色文字 + 细描边。
    - 普通项：透明背景 + 中性文字，hover 时浅底高亮。
- 折叠行为：
  - 折叠后仅保留图标，宽度缩小为 ~80px。
  - Tooltip 可通过浏览器 title 或简单 hover 文案实现，不做复杂交互。

## 五、控件与交互细节

### 按钮

- 主按钮：
  - 渐变背景：`from-emerald-500 to-cyan-500`
  - 文本颜色：白色
  - 圆角：`rounded-xl`
  - 状态：
    - hover：稍微加深渐变色，阴影略增强。
    - disabled：降低不透明度 + `cursor-not-allowed`。

- 次按钮：
  - 边框颜色：`border-slate-300` / 深色模式 `border-slate-600/700`
  - 背景：透明或浅色填充
  - hover：背景略微变浅 / 变深。

### 输入框 / 文本域

- 使用浅色半透明背景 + 细边框 + 大圆角。
- 聚焦状态：使用品牌色描边和柔和的外发光（`focus:ring-emerald-500/50`）。
- 占位符颜色：浅灰（slate-400 ~ slate-500），暗色模式下适当调亮。

### 状态提示

- 错误提示：
  - 背景：`bg-red-50` / 暗色 `bg-red-500/10`
  - 边框：`border-red-200` / 暗色 `border-red-500/20`
  - 文字：`text-red-600` / 暗色 `text-red-400`
- 一般通知 / 提示：
  - 背景：信息型蓝色或品牌绿的浅色块，文字对比度清晰。

## 六、页面级规范

### 登录页

- 视觉目标：第一眼传达「这是一个帮助我搞定简历的专业工具」。
- 结构：
  - 顶部为品牌 Logo 图标（带渐变背景）。
  - 标题：产品名称（中文）+ 简短副标题（简洁说明核心价值）。
  - 表单：邮箱 / 密码，带合理间距。
  - 错误信息：紧贴表单下方的红色提示块。
  - 底部：提供「使用测试账号」按钮及账号信息说明。
- 背景：
  - 使用深色渐变背景，并叠加两侧柔和光圈，营造科技 + 专业感。

### 简历分析页（/chat）

- 布局：左侧侧边栏 + 中间固定宽度输入面板 + 右侧分析结果区域。
- 输入面板：
  - 上方为「简历选择」模块，使用卡片样式、图标 + 说明 + 下拉框。
  - 下方为 JD 文本域，保持较大高度以方便粘贴长文本。
  - 底部为错误 / 提示信息 + 「开始分析」/「停止」按钮区。
- 分析结果：
  - 采用分步骤折叠面板（Accordion），步骤有顺序和图标（emoji）。
  - 当前进行中的步骤在标题旁显示「分析中...」蓝色提示。
  - 已完成步骤显示绿色「✓ 完成」标记。
  - 文本内容使用 `ReactMarkdown` 渲染，配合 `prose` 类优化阅读体验。

### 简历管理页（/resume）

- 布局：侧边栏一致复用；右侧为居中的列表 + 上传区域。
- 简历列表：
  - 每条简历使用卡片样式，左侧图标，中间文件名 + 上传时间，右侧一组操作按钮。
  - 当前用于分析的简历使用品牌绿边框和浅绿背景轻微突出。
- 上传区域：
  - 使用拖拽上传卡片（虚线边框 + 图标 + 说明），点击也可触发文件选择。
  - 上传按钮使用品牌渐变主按钮，旁边配一个返回分析页的次按钮。

## 七、暗色模式

- 整体原则：不是简单反相，而是基于深色背景重新设置对比度。
- 背景偏深、卡片略浅；主文案颜色偏亮，辅助文案使用中度灰。
- 边框在暗色模式下不要太浅，保持一定可见度。
- 渐变按钮和品牌颜色在暗色模式下可基本保持不变，必要时略微调暗饱和度。

## 八、代码风格与实现建议

- 使用 Next.js App Router 和 Tailwind v4 原子类，尽量通过 className 组合完成样式，不在组件内部写复杂自定义 CSS。
- 尽量复用已有的布局模式（如侧边栏、卡片、按钮样式），保证各页面的一致性。
- 对于重复出现的布局模式（如侧边栏 + 顶部 Logo + 菜单），优先抽成复用组件或保持严格的 className 一致。

当你在这个项目中创建或修改任何页面 / 组件（尤其是登录页、简历分析页、简历管理页、模型配置页）时，请严格遵守以上规范，使整体体验保持统一、专业且易于迭代。
